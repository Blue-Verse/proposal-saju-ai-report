핵심 요구사항 분석

  • 관리자 로그인 (폐쇄형 웹) — JWT 기반 인증, 내부 운영진 2~3명 전용
  • 고객 데이터 수급 — 생년월일, 태어난 시간 입력 폼 + MongoDB 저장
  • 사주 API 연동 — KASI 또는 상업용 사주 명리학 API 연동하여 사주 원천 데이터 수급
  • 데이터 정규화 엔진 — 엑셀 If-Then 로직을 Python 기반 엔진으로 이식, 사주 원천 데이터 → 리포트용 구조화 데이터 변환
  • AI 리포트 초안 생성 — LLM(OpenAI GPT-4o / Gemini) 연동, 30페이지/100페이지 두 종류 리포트 자동 생성
  • 웹 에디터 — TipTap 기반 리치 텍스트 에디터, 운영진이 AI 초안을 수정/보완
  • 고해상도 PDF 퍼블리싱 — Puppeteer 기반 30/100페이지 템플릿 PDF 출력, 오행 색상 코딩 반영
  • 관리자 대시보드 — 고객 이력 관리, 리포트 다운로드, 상태 추적

참고 사이트 분석 결과

참고 사이트(min-inter.co.kr)를 분석한 결과, 다음 UX 패턴을 PDF 리포트 템플릿에 반영합니다:

  • 오행(五行) 색상 코딩 — 목(초록), 화(빨강), 토(노랑), 금(흰색), 수(파랑) 컬러 시스템
  • 4기둥 간지 구조 — 연주/월주/일주/시주 데이터 그리드 레이아웃
  • 대운/세운 타임라인 — 시간축 기반 시각화 컴포넌트
  • 다크 테마 + 네온 액센트 — 디자인 참고 (PDF 템플릿에 선택적 반영)

제안 포인트

  1. 사주 분석 앱 직접 개발 경험 — Fortune App에서 사주(四柱) 분석 엔진(LF3) 연동, 생년월일/시간 입력 기반 운세 분석 시스템을 직접 구축한 경험이 있어 사주 도메인에 대한 깊은 이해를 바탕으로 개발합니다.
  2. AI 보고서 자동 생성 시스템 — Series-B 프로젝트에서 ChatGPT 기반 투자심사보고서 자동 생성 시스템을 구축했으며, 동일한 LLM → 에디터 → PDF 파이프라인을 본 프로젝트에 적용합니다.
  3. 7중 PDF 파이프라인 + 리치 에디터 — 170파일 규모의 Lexical 리치 에디터와 7중 PDF 변환 파이프라인 구현 경험을 TipTap + Puppeteer 기반으로 적용합니다.
  4. 참고 사이트 분석 기반 PDF 템플릿 — min-inter.co.kr의 오행 색상 코딩, 4기둥 간지 구조, 대운/세운 타임라인 패턴을 PDF 리포트 디자인에 반영합니다.
  5. LLM 통합 자동화 전문성 — AI Agent 프로젝트에서 멀티 LLM(Claude, Gemini, GPT) 오케스트레이션을 구현하여, LLM 프롬프트 엔지니어링 및 안정적인 AI 연동에 대한 깊은 노하우를 보유하고 있습니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

사주 명리학 전문가가 고객에게 개인 맞춤형 사주 리포트를 제공하기 위한 내부 업무 자동화 시스템입니다. 현재 수작업으로 진행되는 사주 분석 → 리포트 작성 → PDF 출력 과정을 자동화하여 운영 효율을 극대화합니다.

  • 수작업 리포트 작성 시간 대폭 단축 (AI 초안 자동 생성)
  • 일관된 품질의 고해상도 PDF 리포트 출력
  • 고객 데이터 및 리포트 이력의 체계적 관리

3. 기능 요구사항

3.1 관리자 인증 (폐쇄형)

  • [필수] 이메일/비밀번호 로그인
  • [필수] JWT 토큰 기반 세션 관리
  • [선택] 비밀번호 재설정 기능

3.2 고객 데이터 관리

  • [필수] 고객 정보 입력 (이름, 성별, 생년월일, 태어난 시간, 양력/음력)
  • [필수] 고객 목록 조회 (검색, 필터링, 페이징)
  • [필수] 고객 상세 정보 수정/삭제

3.3 사주 API 연동

  • [필수] KASI 또는 상업용 사주 명리학 API 호출
  • [필수] 사주 원천 데이터 수신 및 파싱 (사주팔자, 오행, 십신, 대운, 세운 등)
  • [필수] API 응답 캐싱 (동일 생년월일/시간 재호출 방지)

3.4 데이터 정규화 엔진

  • [필수] 엑셀 If-Then 로직 → Python 기반 규칙 엔진 이식
  • [필수] 사주 원천 데이터 → 리포트용 구조화 데이터 변환
  • [필수] 변환 결과 검증 UI (운영진이 중간 결과 확인 가능)

3.5 AI 리포트 생성

  • [필수] LLM(GPT-4o/Gemini) 프롬프트 기반 리포트 초안 생성
  • [필수] 30페이지/100페이지 두 종류 리포트 템플릿
  • [필수] 생성 진행 상태 표시 (스트리밍 또는 폴링)
  • [선택] 프롬프트 템플릿 관리 (운영진 수정 가능)

3.6 웹 에디터

  • [필수] TipTap 기반 리치 텍스트 에디터
  • [필수] AI 초안 로드 → 운영진 수정/보완
  • [필수] 자동 저장 (드래프트 관리)
  • [선택] 버전 히스토리 (수정 이력 확인)

3.7 PDF 퍼블리싱

  • [필수] Puppeteer 기반 고해상도 PDF 생성
  • [필수] 30페이지/100페이지 PDF 템플릿 (오행 색상 코딩 반영)
  • [필수] PDF 미리보기 기능
  • [필수] PDF 다운로드 (고객별 파일명 자동 생성)

3.8 관리자 대시보드

  • [필수] 고객 리포트 이력 관리 (상태: 입력완료/생성중/편집중/발행완료)
  • [필수] 리포트 다운로드 (PDF 일괄/개별)
  • [선택] 기본 통계 (월별 리포트 생성 수, 고객 수 등)
