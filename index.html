핵심 요구사항 분석

  • 관리자 로그인 — 내부 운영진(2~3명)만 접근 가능한 폐쇄형 웹 환경 구축, JWT 기반 인증
  • 고객 데이터 수급 — 생년월일, 태어난 시간 등 기초 데이터 입력 인터페이스 (양력/음력 변환 포함)
  • 사주 API 연동 — KASI 공공 API 또는 상업용 사주 명리학 API를 통해 기초 간지 데이터 수신
  • 데이터 정규화 엔진 — 클라이언트 엑셀 기반 명리학 If-Then 로직을 시스템에 이식하여 분석 파라미터 산출
  • AI 리포트 초안 생성 — 산출 파라미터를 LLM(Gemini/OpenAI)에 매핑하여 30/100페이지 텍스트 초안 자동 생성
  • 웹 에디터 — AI 생성 초안을 운영진이 직접 수정/보완할 수 있는 리치 텍스트 편집 기능
  • 고해상도 PDF 퍼블리싱 — 디자인 템플릿 기반 자동 조판 + Puppeteer 엔진 고해상도 PDF 출력
  • 관리자 대시보드 — 고객별 리포트 생성 이력 관리, 상태 추적, 최종본 다운로드

제안 포인트

1. 사주 분석 도메인 직접 경험 — 사주(四柱) 분석 엔진 연동, 생년월일/시간 입력 기반 운세 분석 앱을 기획부터 출시까지 풀패키지로 수행한 경험이 있습니다. 사주 명리학의 간지, 오행, 대운/세운 등 도메인 용어와 데이터 구조에 대한 이해를 보유하고 있어, 엑셀 기반 If-Then 로직 이식에 빠르게 대응할 수 있습니다.

2. AI 보고서 자동 생성 시스템 구축 경험 — VC 펀드 관리 플랫폼에서 ChatGPT API를 연동한 투자심사보고서 자동 생성 시스템을 구축했습니다. 입력 데이터를 LLM 프롬프트에 매핑하여 전문적인 보고서를 생성하는 파이프라인 설계에 실전 경험이 있습니다.

3. 7중 PDF 파이프라인 + 리치 에디터 — HTML/이미지/오피스 문서를 PDF로 변환, OCR, 양식 채우기, 병합, 편집까지 처리하는 PDF 파이프라인과 Lexical 기반 170파일 규모의 리치 에디터(26종 커스텀 노드)를 구축한 경험이 있어, 웹 에디터 + PDF 퍼블리싱에 강점이 있습니다.

4. 참고 사이트 분석 반영 — 참고 사이트(min-inter.co.kr)를 분석한 결과, 사주 데이터의 시각적 표현(오행 색상 코딩, 4기둥 구조, 대운/세운 타임라인)에 대한 구체적인 UX 패턴을 파악했습니다. 이를 PDF 템플릿 디자인에 직접 반영하겠습니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

본 프로젝트는 외부 공개용 서비스가 아닌, 운영진 2~3명이 사용하는 내부 업무 자동화 폐쇄형 웹을 구축하는 것입니다. 기존에 수작업으로 진행하던 사주 명리학 리포트 작성 과정을 자동화하여 업무 효율성을 높이는 것이 목표입니다.

파이프라인: 고객 기초 데이터 입력 → 사주 API 연동 → 자체 로직 연산 → LLM 초안 생성 → 웹 에디터 수정 → 고해상도 PDF 발행

3. 기능 요구사항

3.1 관리자 인증 (F-AUTH)

  • [필수] 이메일/비밀번호 기반 로그인
  • [필수] 역할 기반 접근 제어 (운영자/관리자)
  • [선택] 세션 만료 및 자동 로그아웃

3.2 고객 데이터 관리 (F-DATA)

  • [필수] 고객 정보 등록 (이름, 생년월일, 태어난 시간, 성별)
  • [필수] 양력/음력 변환 지원
  • [필수] 고객 목록 조회, 수정, 삭제

3.3 사주 API 연동 (F-API)

  • [필수] KASI 또는 상업용 사주 API 호출 모듈
  • [필수] 기초 간지 데이터 (연주/월주/일주/시주) 수신 및 저장
  • [선택] API 장애 시 fallback 처리

3.4 데이터 정규화 엔진 (F-ENGINE)

  • [필수] 클라이언트 엑셀 기반 명리학 If-Then 로직 시스템 이식
  • [필수] 오행 분석, 용신 판단, 대운/세운 파라미터 산출
  • [선택] 로직 업데이트 시 운영자가 직접 규칙 수정 가능

3.5 AI 리포트 초안 생성 (F-AI)

  • [필수] 산출 파라미터를 LLM 프롬프트에 매핑
  • [필수] 운세, 성격, 직업 추천, 건강, 재물, 대운, 세운 항목 포함
  • [필수] 30페이지 / 100페이지 두 가지 분량 옵션
  • [선택] 프롬프트 템플릿 관리 (운영자가 프롬프트 수정 가능)

3.6 웹 에디터 (F-EDITOR)

  • [필수] 리치 텍스트 편집 (TipTap 기반)
  • [필수] AI 생성 초안 불러오기 및 수정
  • [필수] 자동 저장 및 버전 관리

3.7 PDF 퍼블리싱 (F-PDF)

  • [필수] 디자인 템플릿 기반 자동 조판 (30p / 100p)
  • [필수] Puppeteer 엔진 고해상도 PDF 출력
  • [필수] PDF 미리보기 기능

3.8 관리자 대시보드 (F-DASH)

  • [필수] 고객별 리포트 생성 이력 목록
  • [필수] 상태 관리 (작성중 / 검토중 / 완료)
  • [필수] 최종본 PDF 다운로드

5. 기술 제약사항

  • 내부 운영진(2~3명)만 사용하는 폐쇄형 시스템 — 대규모 트래픽 대응 불필요
  • 클라이언트가 보유한 엑셀 기반 사주 분석 알고리즘을 그대로 시스템에 이식해야 함
  • 리포트에 적용할 마스터 템플릿 가이드라인은 클라이언트가 제공
  • LLM API 비용은 클라이언트 부담 (OpenAI/Gemini 중 선택)

6. 범위 외 사항 (Out of Scope)
